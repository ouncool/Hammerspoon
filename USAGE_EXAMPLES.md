# Hyper Key 使用示例

本文档通过实际场景展示如何使用配置中的各种快捷键。

---

## 场景 1：快速打开浏览器

### 需求
突然需要查阅资料或浏览网页。

### 步骤
1. 按住 **Caps Lock**
2. 同时按 **G**
3. 浏览器立即打开（智能选择已安装的浏览器）

### 结果
✅ 浏览器窗口打开  
✅ 无需从任何应用切换离开  
✅ 比按 Alt+Tab 再选择快得多

---

## 场景 2：快速打开终端并进入项目目录

### 需求
需要在某个 Finder 窗口显示的目录中打开终端。

### 步骤
1. 在 Finder 中打开你的项目目录
2. 按住 **Caps Lock**
3. 同时按 **F**
4. Ghostty（或系统终端）打开，自动进入该目录

### 结果
✅ 终端打开在项目目录  
✅ 无需手动 `cd` 到该目录  
✅ 快速开始工作

**示例**：
```bash
# Finder 显示：~/Projects/my-app
# 按 Hyper+F 后，终端自动进入：
$ pwd
/Users/you/Projects/my-app
```

## 场景 3：在 VS Code 中打开 Finder 目录

### 需求
想在 VS Code 中编辑某个目录中的文件。

### 步骤
1. 在 Finder 中打开包含文件的目录
2. 按住 **Caps Lock**
3. 同时按 **V**
4. VS Code 打开该目录

### 结果
✅ VS Code 打开整个目录为项目  
✅ 文件浏览器自动加载  
✅ 立即可以开始编辑

**示例**：
```
Finder: ~/Desktop/my-document
↓ Hyper+V
VS Code: 打开 ~/Desktop/my-document 作为项目
```

## 场景 4：快速打开终端

### 需求
需要快速打开终端执行命令。

### 步骤
1. 任何时刻、任何应用中
2. 按住 **Caps Lock**
3. 同时按 **T**
4. 终端打开（在默认位置或前台）

### 结果
✅ 终端立即可用  
✅ 无需导航到应用  
✅ 快速执行命令

**示例命令**：
```bash
# 检查 IP
ifconfig

```

## 场景 5：使用窗口管理 - 分屏工作

### 需求
需要在屏幕左右两边各打开一个应用，进行对比或协作。

### 步骤
1. 打开应用 A（例如 VS Code）
2. 按 **Hyper + R** 进入窗口管理模式（看到提示"Window Management Mode"）
3. 按 **H** 使应用 A 贴到左半屏
4. 打开应用 B（例如浏览器）
5. 按 **Hyper + R** 再次进入窗口管理模式
6. 按 **L** 使应用 B 贴到右半屏
7. 按 **Q** 或 **Esc** 退出模式

### 结果
```
┌─────────────────────────────────────┐
│ VS Code (Hyper+R then H) │ Browser │
│         (左半屏)        │ (Hyper+R then L)
│                         │ (右半屏)  │
└─────────────────────────────────────┘
```

## 场景 6：四分屏工作

### 需求
需要在屏幕四个角落各放一个窗口。

### 步骤
对于每个应用：
1. 按 **Hyper + R** 进入窗口管理模式
2. 按对应的角位键：
   - **Y** → 左上
   - **U** → 左下
   - **I** → 右上
   - **O** → 右下
3. 按 **Q** 退出

### 结果
```
┌──────────────┬──────────────┐
│      Y       │      I       │
│    (应用1)    │    (应用2)    │
├──────────────┼──────────────┤
│      U       │      O       │
│    (应用3)    │    (应用4)    │
└──────────────┴──────────────┘
```

## 场景 7：最大化窗口

### 需求
快速全屏当前应用。

### 步骤
1. 按 **Hyper + R** 进入窗口管理模式
2. 按 **F**（Maximize）

4. 按 **Q** 退出

### 结果

✅ 最大化工作空间  
✅ 一键全屏

## 场景 8：关闭窗口

### 需求
快速关闭当前窗口。

### 步骤
1. 按 **Hyper + R** 进入窗口管理模式
2. 按 **C**（Close）
3. 窗口关闭
4. 自动退出管理模式

### 结果
✅ 窗口关闭  
✅ 比 Cmd+W 或 Cmd+Q 更统一的操作

## 场景 9：重新加载配置

### 需求
修改了配置文件后，需要重新加载。

### 步骤
1. 修改任何配置文件（例如 `config.lua` 或 `hyper-key.lua`）
2. 保存文件
3. 按 **Cmd + Ctrl + Alt + R**
4. 看到通知"✅ Hammerspoon 配置已加载"

### 结果
✅ 新配置立即生效  
✅ 无需重启应用  
✅ 快速迭代开发

## 场景 10：强制粘贴

### 需求
某些网站禁用了粘贴功能（例如密码框），需要绕过。

### 步骤
1. 复制你的内容：**Cmd + C**
2. 点击目标输入框
3. 按 **Cmd + Shift + V**（而不是 Cmd + V）
4. 内容粘贴成功

### 结果
✅ 绕过网站的粘贴限制  
✅ 可以粘贴密码、代码等  
✅ 节省手动输入的时间

## 场景 11：日常工作流程

### 典型的工作日开始

```
早晨：
1. 按 Hyper+T 打开终端
   → 查看邮件：mutt
   
2. 打开 Finder 到项目目录

3. 按 Hyper+V 在 VS Code 打开项目
   → 开始编写代码

4. 按 Hyper+R 分屏
   → H 键 VS Code 放在左半屏
   → L 键浏览器放在右半屏
   → I 键终端在右上
   
5. 按 Hyper+F 在新终端中打开项目目录
   → 运行测试：npm test

6. 按 Hyper+G 打开浏览器查看文档

7. 工作中断时按 Hyper+R → F 全屏当前应用
   → 专注模式
```

## 提示和技巧

### ✨ 提示 1：组合使用快捷键
```
场景：需要在新终端中运行项目代码
1. Hyper+T 打开终端
2. Finder 中打开项目目录
3. Hyper+F 在该目录打开新终端

```

### ✨ 提示 2：快速应用切换
```
比 Alt+Tab 更快：
1. 关闭当前应用：Hyper+R then C
2. 打开新应用：Hyper+T / Hyper+G / Hyper+V
3. 使用窗口管理：Hyper+R then H/L
```

### ✨ 提示 3：高效开发
```
多屏工作流：
1. 左屏：编辑器（Hyper+V from Finder）
2. 右屏：浏览器（Hyper+G）
3. 下屏：终端（Hyper+F from Finder）
4. 按 Hyper+R → Tab 显示帮助
```

### ✨ 提示 4：修改配置后重载
```
快速迭代：
1. 编辑配置
2. Cmd+Ctrl+Alt+R 重载
3. 立即测试新快捷键
4. 无需重启 Hammerspoon
```

## 常见问题

**Q: 为什么有时快捷键不响应？**  
A: 检查 Hyper 键是否正确配置在 Karabiner-Elements 中。按 `Hyper+R` 会显示"Window Management Mode"，表示 Hyper 键有效。

**Q: 可以修改快捷键吗？**  
A: 完全可以。在对应模块中修改按键字母即可。

**Q: Caps Lock 完全失效了吗？**  
A: 是的。Caps Lock 现在完全映射为 Hyper 键，不再有大小写切换功能。这是设计特性。

**Q: 这会影响其他应用吗？**  
A: 不会。Hammerspoon 的快捷键仅在需要时激活。
